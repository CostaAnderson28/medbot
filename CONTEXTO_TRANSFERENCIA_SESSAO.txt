MEDBOT - CONTEXTO COMPLETO PARA CONTINUAR EM OUTRA MAQUINA
Data de geracao: 2026-04-09

1) RESUMO EXECUTIVO
- Projeto: chatbot medico com painel web, API em Express e SQLite (better-sqlite3).
- Integracoes: Instagram webhook (Meta), Claude (Anthropic), autenticacao JWT.
- Deploy: VPS Hostinger com PM2 + Nginx.
- Problema principal da sessao: dashboard sem carregar dados em producao.
- Causa raiz principal encontrada: query SQL de engajadas em src/routes/metrics.js com aspas incorretas (quebrava parsing/execucao).
- Estado final reportado: app online, health OK, logs antigos ainda mostravam erro anterior (nao necessariamente erro atual).

3) EVENTOS IMPORTANTES DA SESSAO (RETROSPECTO)
- Foi identificado que a VPS tinha mais de um app no PM2 (chat-bot antigo + medbot).
- Em alguns momentos, chat-bot ocupava porta 3000, causando conflito de deploy.
- Houve falha nativa do better-sqlite3 na VPS (libnode/make/build tools), exigindo rebuild e/ou ferramentas de compilacao.
- Dashboard local e producao foram confundidos em alguns momentos; no fim ficou claro:
  - local estava bom
  - producao estava com versao/estado inconsistente ate ajustar pull/restart/codigo
- Erro SQL visto no terminal:
  - SqliteError: no such column: "user"
  - Indica uso incorreto de role="user" em vez de string literal adequada

4) CORRECAO DE CODIGO CRITICA (METRICS)
Arquivo alvo: src/routes/metrics.js

Ponto critico:
- Query de engajadas precisava evitar quebra por aspas no JavaScript e no SQL.

Forma segura recomendada:
engajadas: q("SELECT COUNT(DISTINCT c.id) as count FROM conversations c WHERE c.doctor_id=? AND (SELECT COUNT(*) FROM messages WHERE conversation_id=c.id AND role='user') > 1"),

Observacao:
- Evitar colocar SQL com role='user' dentro de string JS com aspas simples externas sem escape correto.

6) COMANDOS UTEIS DE OPERACAO (PRODUCAO)
A) Atualizar codigo no servidor
- cd /opt/medbot
- git pull origin main
- pm2 restart medbot

B) Ver status/logs
- pm2 list
- pm2 logs medbot --lines 50 --nostream
- curl -s http://localhost:3000/health

C) Limpar logs antigos para nao confundir diagnostico
- pm2 flush
- pm2 restart medbot
- pm2 logs medbot --lines 20 --nostream

D) Se better-sqlite3 quebrar por build nativo
- apt update && apt install -y build-essential python3-dev
- cd /opt/medbot
- npm rebuild better-sqlite3
- pm2 restart medbot
